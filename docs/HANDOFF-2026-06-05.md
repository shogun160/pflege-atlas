## Zu fixen in der nächsten Session

### A) Aus dem Final-Code-Review (Reviewer-Verdict: „conditionally approve")

**A1 — Critical: Dual-Status-Konflikt in Articles**
`src/collections/Articles.ts` hat sowohl `status` (custom: `draft/in_review/published/archived`) als auch Payload-natives `versions.drafts: true` (eigenes `_status`-Feld `draft/published`). Beide laufen unabhängig — Editor publishen ≠ Payload-Draft veröffentlichen. Public-Access-Rule gated nur auf `status`. Vorschlag: **`versions: { drafts: false, maxPerDoc: 50 }`** für V1, History bleibt erhalten. Oder klarer Kommentar im Code, der beschreibt, dass `status` der einzige Visibility-Gate ist.

**A2 — Important: a11y-Regression im ArticleTOC (Mobile)**
`src/components/ArticleTOC.tsx:76` setzt `aria-hidden="true"` auf den ausgeklappten Mobile-Content. Screenreader sehen den ganzen aufgeklappten TOC nicht. Fix: `aria-hidden`-Attribut entfernen.

**A3 — Important: A-Z-Index gruppiert Umlaute fälschlich unter `#`**
`src/app/(frontend)/index/page.tsx:17-18`: `/[A-Z]/.test(letter)` ist `false` für `Ä/Ö/Ü`. Artikel wie „Ärztliche Anordnung" landen unter `#`. Fix: Regex auf `/[A-ZÄÖÜ]/` erweitern und Umlaute beim Grouping zu Basisbuchstabe mappen (Ä→A, Ö→O, Ü→U), Section-Heading aber als „A" anzeigen (und „Ärztliche Anordnung" unter `A` einsortieren).

**A4 — Important: CI fehlt `pnpm lint`-Step**
`.github/workflows/ci.yml`: zwischen `pnpm install --frozen-lockfile` und `pnpm payload migrate` einfügen: `- run: pnpm lint`.

### C) Aus Browser-Verifikation (Oliver, 2026-06-05)

**C1 — Glossar/Volltext-Index auf der Startseite fehlt**
Aktuell findet ein Nutzer nur über Intent-Karten oder direkte URL Artikel. Es braucht zusätzlich eine prominente Möglichkeit, **alle Artikel** zu durchstöbern. Aktuelle `/index`-Seite ist ein A–Z-Index, aber von der Startseite nicht erreichbar. Vorschlag:
- Link „Alle Artikel A–Z" prominent auf der Startseite (z. B. unter den Intent-Karten oder neben der Suche)
- Optional: kleine Stichwort-Wolke der häufigsten Themen
- Mittelfristig: echte Suche (Meilisearch, eigener Folge-Plan)

**C2 — React Hydration Error im Browser**
Beim Öffnen eines Artikels in DevTools:
```
A tree hydrated but some attributes of the server rendered HTML didn't match the client properties.
```
Mögliche Ursachen (in unserer Codebase):
- ArticleTOC ist `'use client'` mit `useState(false)` — `open`-State könnte mismatchen, wenn jemals server-side anders gerendert wird
- Date-Formatting im `reviewedAt`-Feld (`toLocaleDateString('de-DE')`) könnte zwischen Server- und Client-Locale abweichen — aber Testartikel haben kein `lastReviewedAt`
- Inline JSON-LD `<script>` mit `dangerouslySetInnerHTML` — falls die Daten zwischen Renders abweichen
- Browser-Extension (Reviewer-Hinweis) — sollte ausgeschlossen werden

Debug-Pfad für die nächste Session: DevTools → Components-Tab → roten Stack-Trace lesen, identifiziertes Element isolieren, ohne Extensions im Inkognito-Modus testen.

**C3 — Optik überarbeiten (visueller Bug + Design-Refresh nötig)**

*Akuter Bug (Screenshot vom Artikel-Detail):*
- **Schwarzer Balken** überlagert die untere Hälfte der Seite. Ursache: die vom Payload-CLI generierte `src/app/(frontend)/styles.css` enthält noch `html { background: rgb(0, 0, 0); }`. Beim Task-3-Tailwind-Setup wurden Body-Farben überschrieben, aber die `html`-Regel blieb stehen. Wenn `body` nicht die volle Höhe abdeckt (z. B. weil `flex-1` nicht greift), scheint das Schwarz darunter durch. Fix: `html`-Background-Regel in `styles.css` löschen oder auf `#fff` setzen.
- **Riesige H1-Schriftgröße** (Test-Artikel-Titel füllt halbe Seite). Vermutlich ebenfalls Erbe der CLI-CSS-Regeln (`html { font-size: 18px; line-height: 32px }` + Tailwind Typography `prose` greifen sich gegenseitig an).

*Design-Refresh (parallel angehen):*
