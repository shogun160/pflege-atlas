# PflegeAtlas Visual Polish V1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the PflegeCommons placeholder identity with the final PflegeAtlas brand, install a token-based design system on Tailwind v4, swap in IBM Plex Serif/Sans webfonts, and polish the reading-relevant components (Wordmark, Header, Favicon, Article page, TOC, Disclaimer, IntentCards, Footer, Homepage).

**Architecture:** Tailwind v4 `@theme` block in `src/app/(frontend)/styles.css` defines six color tokens, three font-family variables, and the Plex font stack. `next/font/google` self-hosts IBM Plex Serif and Plex Sans and exposes them as CSS variables on the `<html>` element. A new `<Wordmark>` component renders the `Pflege·Atlas` mid-dot wordmark in three sizes. Tailwind Typography is themed via plugin config to use Plex Serif for headings and Plex Sans for body inside `prose` blocks. Existing components (`Footer`, `IntentCards`, `ArticleTOC`, `ArticleDisclaimer`, `ArticleLayout`, `page.tsx`, `artikel/[slug]/page.tsx`) are reworked to consume the tokens and the new typography.

**Tech Stack:** Next.js 16 (App Router), Tailwind CSS v4 with `@theme`, `@tailwindcss/typography`, `next/font/google`, IBM Plex Serif + Plex Sans, Vitest 4 + React Testing Library for component tests, Payload CMS 3.85 untouched.
