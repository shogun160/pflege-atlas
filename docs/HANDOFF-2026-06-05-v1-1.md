## Latente / aufgeschobene Findings

Aus den Reviews während Tasks 3–11 aufgekommen. **Bewusst nicht in V1.1 gefixt** — Kandidaten für V1.2:

- **Spec ↔ Code Drift in Wordmark (Task 3).** Spec sagt Mid-Dot in Weight 700 und Desktop-Größe 24 px; Code erbt 500 (`font-medium`) und `text-xl` (20 px). Bewusst plan-treu umgesetzt — vor Merge entscheiden, ob Spec relaxen oder Code nachziehen.
- **`tracking-[0.08em]` ist als Section-Label-Recipe 5× im Code dupliziert** (Article-Page Intent-Label, ArticleTOC × 3, IntentCards). Empfehlung: in einer V1.2-Iteration entweder `--tracking-label: 0.08em;` als Tailwind-Theme-Eintrag plus `tracking-label`-Utility, oder ein `<SectionLabel>`-Helper-Component, das die vier Klassen kapselt.
- **IntentCards `accent` + `paid?` Redundanz (Task 9).** Das `accent`-Feld trägt nur noch zwei Werte (`border-brand` / `border-ink-muted`) und wäre als `intent.paid ? '...' : '...'` direkt aus `paid?` ableitbar. Refactor-Kandidat für V1.2.
- **IntentCards a11y Double-Announce.** `<section aria-label="Ich brauche…">` plus sichtbares `<p>` mit gleichem Text — Screenreader liest doppelt. Fix: `aria-labelledby` auf das `<p>` setzen statt `aria-label` am `<section>`.
- **Search-Input ist Stub** — `<form action="/suche" method="get">` läuft auf eine nicht existierende Route. Wird mit dem Meilisearch-Plan funktional. Visuell-polierte Suche signalisiert Funktion, die noch fehlt; bei längerem Aufschub Meilisearch-Plan ggf. Input als `aria-disabled` markieren.
- **Favicon-URL unter Next 16 + Turbopack ist `/icon-<hash>.svg`** statt plain `/icon`. Browser-`<link rel="icon">`-Resolution funktioniert; Plan-Step-3-Smoketest (`curl /icon`) liefert 404, ist aber keine echte Regression — Next hashed das SVG für Cache-Busting.
- **`tests/unit/schema-org.test.ts`** verwendet noch `pflegecommons.de` als Test-Fixture, und `package.json "name"` heißt noch `pflegecommons`. Beides ist Teil eines separaten Repo-/Package-Rename-Plans, **nicht** Teil von V1.1.
- **Latente Tech-Debt aus früheren Handoffs (unverändert offen):** Datum-Locale-Hardening in `artikel/[slug]/page.tsx:59` (`toLocaleDateString` Node-vs-Browser-Drift), JSON-LD-`createdAt`/`updatedAt` `Date|string`-Coercion, `6× as any` in Article-Page, doppelte Payload-Query pro Artikel-Render (mit React `cache()` deduplizieren), GitHub-Link im Footer ist bare-Placeholder.
